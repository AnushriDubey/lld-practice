Design Pattern Practice Questions 🎯
I'll give you scenarios, and you tell me which design patterns to use!

🎮 Question 1: Notification System
Scenario: You're building a notification system for an e-commerce platform. When an order is placed:
	• Send email notification
	• Send SMS notification
	• Send push notification
	• Update dashboard
	• Log to analytics
In the future, you might add:
	• Slack notifications
	• Discord notifications
	• Webhook calls
Requirements:
	• Easy to add new notification types
	• Notifications should be sent independently (one failure shouldn't affect others)
	• Some users might want only email, some only SMS, some both
Question: Which design pattern(s) would you use and why?

🎮 Question 2: Payment Gateway Integration
Scenario: Your app needs to support multiple payment gateways:
	• Stripe
	• PayPal
	• Razorpay
	• Square
Each gateway has different APIs:
java
// Stripe
StripeClient.charge(amount, cardToken)
// PayPal
PayPalAPI.createPayment(amount, currency, payerId)
// Razorpay
RazorpaySDK.processPayment(orderAmount, customerDetails)
```
**Requirements:**
- Client code should work with any gateway
- Easy to switch between gateways
- Easy to add new gateways
- Client shouldn't know gateway-specific details
**Question:** Which design pattern(s) would you use?

## **🎮 Question 17: Request Processing Pipeline**
**Scenario:**
Web application with request handling chain:
1. Authentication filter
2. Authorization filter
3. Logging filter
4. Rate limiting filter
5. Actual request handler
Each handler:
- Can process request
- Can pass to next handler
- Can stop chain and return response
**Requirements:**
- Process request through multiple handlers
- Handler can decide to pass or stop
- Easy to add/remove handlers
- Order matters
**Question:** Which design pattern(s) would you use?
